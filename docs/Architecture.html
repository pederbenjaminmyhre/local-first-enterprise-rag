RAG Pipeline Flow

User Question
→
Embed via Ollama
→
Hybrid Search (SQL)
→
Build Prompt
→
Generate Answer (LLM)
  1. The user's natural-language question is converted to a 1024-dimensional vector via mxbai-embed-large
  2. The vector is passed to a T-SQL stored procedure that combines VECTOR_DISTANCE (cosine similarity) with optional metadata filters (category, color)
  3. The top 5 most relevant products and the original question are composed into a prompt
  4. The prompt is sent to llama3.2 for natural-language answer generation (streamed token-by-token in the WPF app)

Key Architectural Decisions

ADO.NET over Entity Framework

Raw Microsoft.Data.SqlClient is used for all database access. This showcases T-SQL and stored procedure expertise — a deliberate choice for a portfolio targeting enterprise data architect roles. An ORM would hide the SQL Server-specific features (VECTOR types, VECTOR_DISTANCE, DiskANN) that are the project's key differentiators.

SQL Server 2025 as the Vector Store

Rather than deploying a separate vector database (Pinecone, Weaviate, Milvus), embeddings are stored directly in the DimProduct table alongside relational data. This eliminates an entire infrastructure component, simplifies governance, and keeps all data in a single engine with unified backup, security, and access control.

WPF for Local-First, Web for Cloud

The desktop WPF application emphasizes the air-gapped, on-premise nature of the system — it is not a browser application that implies cloud dependency. The ASP.NET Minimal API web project is a thin adapter that reuses 100% of Core and Infrastructure, proving the same codebase scales to cloud deployment without modification.

Hybrid Portability Pattern

The system uses environment-based configuration to switch between local and Azure deployments:

  • Local: SQL auth (sa password in Docker), Ollama on localhost:11434
  • Azure: Connection string via App Settings, Ollama via Container App HTTPS endpoint

No code changes are required — only configuration values differ between environments.

Test Strategy

Unit tests mock all external dependencies (HTTP handlers, SQL connections) and verify pipeline wiring, prompt construction, and error handling. Integration tests use IAsyncLifetime for async service detection and TestPriorityAttribute ordering to minimize Ollama model swaps (embedding tests run before generation tests — one swap instead of many).
