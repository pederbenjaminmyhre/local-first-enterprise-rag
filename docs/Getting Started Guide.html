Getting Started Guide

Prerequisites

  • .NET 9 SDK — dotnet --version should return 9.x or higher
  • Docker Desktop — for running SQL Server 2025 locally
  • Ollama — installed natively on Windows for local LLM inference

Initial Setup (One-Time)

1. Start Docker and SQL Server

docker compose up -d

This starts SQL Server 2025 and Nginx. Wait until the health checks pass:

docker ps   # All containers should show "healthy"

2. Restore the Database

Download the AdventureWorksDW2022 backup and place it at infrastructure/scripts/AdventureWorksDW2020.bak, then run:

docker exec rag-sql-server bash /scripts/init-db.sh

This restores the database and applies all four SQL migration scripts (vector column, stored procedure, index).

3. Pull Ollama Models

ollama pull mxbai-embed-large
ollama pull llama3.2:1b

4. Run the Ingestion Pipeline

cd src/EnterpriseRag.Ingestion
dotnet run

This embeds all product descriptions into the database. Takes approximately 5 minutes on first run. Safe to re-run — it only processes products with missing embeddings.

Using the Desktop Application (WPF)

Launch

dotnet run --project src/EnterpriseRag.Desktop

Use Case 1: Basic Product Search

  1. Type a natural-language question in the search box, e.g., "What mountain bikes do you have?"
  2. Click Search (or press Enter)
  3. The Sources panel populates within seconds showing the top 5 most relevant products with similarity scores
  4. The AI Response panel streams the LLM's answer token by token

Use Case 2: Filtered Search

  1. Select a category from the Category dropdown (e.g., "Road Bikes")
  2. Optionally select a color from the Color dropdown (e.g., "Red")
  3. Type your question: "Which is the lightest option?"
  4. Click Search — results are filtered to only products matching your category and color

Use Case 3: Comparing Products

  1. Ask a comparative question: "Compare your touring bikes for long-distance trips"
  2. The AI response synthesizes information from multiple source products
  3. Review the Sources panel to see the raw product data and similarity scores
Tip: The status bar at the bottom shows timing information — how many products were retrieved and how long the AI generation took.

Use Case 4: Cancel a Long Query

If the AI response is taking too long, click the Cancel button that appears during search. This cancels the LLM generation immediately.

Using the Web Application (Azure)

The web version is available at your Azure deployment URL. The interface mirrors the desktop application:

  1. Open the URL in any browser
  2. Type a question in the search box
  3. Select optional category and color filters
  4. Click Search
  5. The AI response and sources appear in the same layout as the desktop version
Note: The Azure deployment uses CPU-only Ollama with scale-to-zero. The first query after idle may take 30–60 seconds as the container wakes up. Subsequent queries respond faster while the container is warm.

Running Tests

# Unit tests (no services required)
dotnet test tests/EnterpriseRag.Tests.Unit

# Integration tests (requires Docker + Ollama running)
dotnet test tests/EnterpriseRag.Tests.Integration

# All tests
dotnet test src/EnterpriseRag.slnx
