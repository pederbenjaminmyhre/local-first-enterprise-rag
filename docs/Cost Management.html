Cost Optimization Strategies

1. Scale-to-Zero Container App

The Ollama LLM container is configured with minReplicas: 0 and maxReplicas: 1. When no queries are being processed, the container shuts down entirely and incurs zero compute charges. This is the single largest cost saving — a continuously running GPU container costs $100–$300/month, while our scale-to-zero configuration charges only for actual inference time.

  • Trade-off: First query after idle takes 30–60 seconds (cold start + model loading)
  • Mitigation: For demos, send a warm-up query before the presentation
  • Production note: Set minReplicas: 1 during business hours if consistent response time is required

2. Azure SQL Database Basic Tier

The Basic tier ($4.99/month) provides 5 DTUs and 2 GB storage — more than sufficient for the 25-product demo dataset with vector embeddings. The full AdventureWorksDW2020 dataset (606 products) would fit comfortably within this limit.

Savings vs. SQL Managed Instance: The original CLAUDE.md plan specified Azure SQL Managed Instance ($350+/month). By confirming that Azure SQL Database Basic tier supports the VECTOR data type and VECTOR_DISTANCE function, we reduced database costs by 98%.

3. Free App Service Tier

The F1 (Free) App Service tier hosts the .NET 9 web application at zero cost. Limitations include no custom domain, no SSL certificate, and shared compute — all acceptable for a portfolio demo. Upgrading to B1 ($13/month) adds custom domains and dedicated compute when needed.

4. Eliminating External AI API Costs

By running Ollama locally and in Azure Container Apps, the system avoids all per-token API charges from OpenAI, Azure AI, or other LLM providers.

Local Development: Zero Cloud Cost

The local-first design means all development, testing, and demonstration can occur entirely on a developer's machine using Docker Desktop and native Ollama — at zero cloud cost. Azure is only needed for the "enterprise scalability" proof, not for day-to-day development or demonstrations.

Cost Monitoring Best Practices

  • Set Azure budget alerts — configure a $25/month budget with email alerts at 50%, 80%, and 100%
  • Monitor Container App replica count — ensure scale-to-zero is functioning by checking that replicas drop to 0 after idle timeout
  • Review DTU utilization — if the Basic tier's 5 DTUs become a bottleneck during demos, upgrade to S0 ($15/month) only when needed, then downgrade afterward
  • Shut down when not demoing — for maximum savings, stop the Container App entirely between demo sessions: az containerapp update --name enterpriserag2-ollama --resource-group enterpriserag2-rg --min-replicas 0 --max-replicas 0
  • Use the health endpoint — /api/health shows whether Ollama is running, helping you verify scale-to-zero is active before being billed
Important: Container App models (llama3.2, mxbai-embed-large) are stored in ephemeral storage. If the container restarts after scaling to zero, models must be re-pulled (~3 minutes). For a scheduled demo, warm up the container 5 minutes before by hitting the health endpoint.
