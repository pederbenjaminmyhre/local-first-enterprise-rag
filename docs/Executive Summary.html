Executive Summary

Enterprise adoption of AI-powered search and question-answering is accelerating, yet two persistent barriers block regulated industries from deploying these systems: data privacy and recurring cost. Organizations in Finance, Healthcare, and Defense cannot send proprietary data to third-party AI providers, and the per-token pricing of cloud LLM APIs creates unpredictable, escalating operational expenses.

This project solves both problems simultaneously. It delivers a production-ready Retrieval-Augmented Generation (RAG) system that operates entirely within the organization's security perimeter while eliminating external AI token costs.

The Business Problem

Traditional AI search solutions require organizations to transmit sensitive product, customer, or operational data to external APIs (OpenAI, Azure AI, etc.). This creates three risks:

  • Compliance exposure — regulated data leaves the firewall, violating SOC 2, HIPAA, or ITAR requirements
  • Cost unpredictability — per-token billing scales linearly with usage, often reaching $5,000–$20,000/month for enterprise workloads
  • Vendor lock-in — switching AI providers requires re-architecting the entire search pipeline

The Solution

Local-First Enterprise RAG combines three technologies into a single, self-contained system:

  • SQL Server 2025 native vector search — stores embeddings alongside relational data in the same governed database, eliminating the need for a separate vector database
  • Ollama local LLM — runs AI models on-premise with zero data leaving the network
  • .NET 9 / WPF desktop application — provides a polished user interface for natural-language product search

Business Value Delivered

100% Air-Gapped $0 Token Cost ~$10/mo Azure

  • Data sovereignty — all data, models, and inference remain inside the organization's firewall
  • Zero recurring AI costs — local LLM inference replaces per-token API billing entirely
  • Hybrid portability — the same codebase runs on a developer laptop or scales to Azure with no code changes
  • Reduced infrastructure — SQL Server 2025's built-in vector search eliminates Pinecone, Weaviate, or other dedicated vector database subscriptions
  • Future-proof — built on the Microsoft data platform roadmap (SQL Server 2025 + .NET 9 + Azure), ensuring long-term vendor support

Who This Serves

This system is designed for organizations that need AI-powered search but operate under strict data governance: financial services firms querying internal product catalogs, healthcare providers searching clinical documentation, and defense contractors working within classified networks. The architecture proves that enterprise AI does not require sacrificing security or budget.
