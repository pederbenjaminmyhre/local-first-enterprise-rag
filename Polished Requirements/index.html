Local-First Enterprise RAG

Project Requirements, Architecture & Career Outreach Strategy

Target Environment: Windows 11 / Server 2025  |  .NET 9  |  Docker Desktop
Primary Objective: Build a production-ready, air-gapped Retrieval-Augmented Generation system using SQL Server 2025's native vector capabilities and local LLM orchestration.

1.1 Architectural Vision

This project follows a Hybrid Portability philosophy. It is developed Local-First to prove data sovereignty and zero-cloud dependency, then deployed to a Unified Microsoft Stack (Azure + SQL Server 2025) to demonstrate enterprise-grade scalability. The result is a single codebase that runs identically on a developer laptop and in a multi-tenant cloud environment.

1.2 Value Proposition

"I bridge the gap between legacy SQL infrastructure and modern AI, specializing in high-security, low-cost, on-premise AI deployments."

With 23+ years of enterprise SQL Server experience, this project demonstrates a pragmatic approach to AI adoption: one that respects data governance, eliminates recurring cloud-token costs, and extends existing infrastructure rather than replacing it.

2.2 Vector Store Design

The vector store is built directly into the relational data model by extending the existing AdventureWorksDW2020.DimProduct table with a VECTOR(1024) column named DescriptionVector. This approach avoids a separate vector database, keeping all data—relational and semantic—in a single, governable engine.

2.3 Ingestion Pipeline

Batch Embedding Service

A dedicated C# service iterates through the EnglishDescription field of each product record, calls the local Ollama embedding API (mxbai-embed-large), and performs a bulk UPDATE to populate the DescriptionVector column. This pipeline is idempotent and can be re-run as product data changes.

2.4 Hybrid Search

Search combines semantic similarity with metadata filtering in a single T-SQL stored procedure:

  • Vector Distance: The VECTOR_DISTANCE function (Cosine Similarity) ranks products by semantic relevance to the user's query embedding.
  • Metadata Filter: Standard LIKE or CONTAINS predicates narrow results by category, color, or other structured attributes.

This hybrid approach leverages 20+ years of T-SQL indexing expertise alongside modern vector search—proving that AI enhances the relational database rather than replacing it.

  1. The user submits a natural-language question through the WPF interface.

3. Deployment & Hosting Strategy

The deployment follows a Single-Cloud, Multi-Tier approach on Microsoft Azure, leveraging the same T-SQL engine used during local development to minimize migration friction and eliminate the "learning tax" of switching platforms.

3.1 Azure Hosting Matrix

Security Proxy Nginx (Sidecar Container) HTTPS bridge between SQL Server and the Ollama REST API

3.2 Phase A — Local-First Development Sovereign Proof

Objective

Prove the system operates entirely air-gapped—no data leaves the developer's machine. All services (SQL Server 2025, Ollama, Nginx) run in Docker Desktop containers on the local workstation.

Deliverable

A fully functional .NET 9 WPF application scaffolded with Claude Code, connected to a local SQL Server 2025 container, demonstrating end-to-end RAG with zero external dependencies.

3.3 Phase B — Cloud-Native Staging Enterprise Proof

Objective

Demonstrate the ability to scale the identical codebase to thousands of concurrent users on Azure infrastructure.

Deliverable

A GitHub Actions CI/CD pipeline that, on push to main, automatically deploys the application to Azure App Service and applies schema migrations to Azure SQL Managed Instance.

4. Cloud Architecture Decisions

The following architectural decisions are specifically selected to resonate with enterprise security reviewers, hiring managers, and technical leadership evaluating this portfolio.

4.1 DiskANN Optimization

Decision: SQL Server 2025 was chosen specifically for its DiskANN algorithm (developed by Microsoft Research), which maintains 95% recall accuracy while searching datasets larger than available RAM. This eliminates the need for specialized vector databases like Pinecone, significantly reducing infrastructure cost and operational complexity.

4.2 Privacy-First Guardrails

Decision: Ollama is hosted in a private Azure Container, ensuring that corporate data used for retrieval-augmented generation is never exposed to public AI training pipelines. Unlike OpenAI's default terms of service, this architecture guarantees that sensitive enterprise data remains within the organization's security boundary.

4.3 Zero-Trust Identity (Managed Identity)

Decision: The C# application connects to SQL Server using Azure Managed Identities, eliminating all passwords and connection strings from source code. This is a critical security posture for regulated industries and represents a significant green flag for Security Architects during portfolio review.
